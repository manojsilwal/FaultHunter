# FaultHunter feature test specification

FaultHunter evaluates TradeTalk HTTP surfaces against independent benchmarks. This document
describes the evaluation architecture and lists every `TestCase` (authoritative source: Python
[`case_bank.py`](faulthunter/case_bank.py)).

## Evaluation pattern (agent-architect framing)

- **Workflow**: treat FaultHunter as a **loop** — capture the target response, pull external
  reference data, compare and judge until each case is scored.
- **Target**: TradeTalk via the HTTP adapter (`faulthunter/targets/tradetalk.py`).
- **External toolbox (“MCP” analogy)**:
  - **Yahoo / `yfinance`**: numeric benchmarks and parity fields for supported cases.
  - **Optional browser probe**: Playwright snippet from a public quote URL for supplemental evidence
    (not a full site walkthrough).

## Run profiles

| Profile | Scope |
|---------|--------|
| `smoke` | Core cases; fast feedback. |
| `daily` | Smoke cases plus additional daily coverage. |
| `deep` | Currently identical to `daily` in `build_case_bank` until extended. |
